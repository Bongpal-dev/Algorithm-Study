## 병합 정렬 (Merge Sort)

- 병합 정렬은 대표적인 분할 정복 알고리즘의 하나로, 주어진 배열을 최소 단위까지 분할한 뒤 다시 합치면서 정렬하는 알고리즘이다.

- 정렬을 할때 데이터 크기만큼의 추가공간이 필요하기 때문에 제자리 정렬이 될 수 없으나, 공간복잡도는 O(n)으로, 다른 알고리즘들과 크게 차이나지 않는다.

<br>

#### 분할 정복 알고리즘 (Divide and conquer)

- 하나의 큰 문제를 여러개의 작은 문제로 분할하여 해결하는 방식의 알고리즘을 말한다.

- 즉, 한번에 여러가지를 해결해야 하는 문제를 작게 분할해 각각의 문제를 해결하고 그 결과를 이용해 전체 문제를 해결하는 방식이다.

<br>

## 정렬방식

정렬해야하는 배열을 더 이상 나눌 수 없을 때까지 나눈뒤 해당 배열의 원소들을 비교해 정렬하며 합친다.

예를들어 '2, 4, 1, 5, 6, 7, 3, 9'의 배열이 있을 경우 다음과 같이 배열이 분할 된다.

```kotlin
intArrayOf(2, 4, 1, 5, 6, 7, 3, 9) 

intArrayOf(2, 4, 1, 5 | 6, 7, 3, 9) 

intArrayOf(2, 4 | 1, 5 | 6, 7 | 3, 9) 

intArrayOf(2 | 4 | 1 | 5 | 6 | 7 | 3 | 9) // 각각의 배열을 최소단위(1개의 원소만 가지고있는 경우)까지 분할
```

이후 분할된 배열들을 합치면서 값을 비교해 정렬한다.

```kotlin
 intArrayOf(2 | 4 | 1 | 5 | 6 | 7 | 3 | 9)

 intArrayOf(2, 4 | 1, 5 | 6, 7 | 3, 9)
// [2, 4]와 [1, 5]가 합쳐지면서
// [1] -> [1, 2] -> [1, 2, 4] -> [1, 2, 4, 5] 의 순서로
// 각 원소의 크기르 비교하며 합쳐짐

intArrayOf(1, 2, 4, 5 | 3, 6, 7, 9) 

intArrayOf(1, 2, 3, 4, 5, 6, 7, 9) // 이후에도 크기를 비교하며 합쳐지는것이 동일하다.
```

데이터를 분할해서 처리하기때문에 O(nlogn)의 시간복잡도를 가지고 있다.
